What is Regular Expression?

A regular expression (regex) is a powerful pattern matching tool that lets you search for complex text patterns instead of just exact matches.

Basic Examples:

^_ab
Starts with "_ab" (the ^ means beginning)
ck$
Ends with "ck" (the $ means end)
.*token.*
Contains "token" anywhere (the .* means any characters)
(akamai|datadome)
Matches "akamai" OR "datadome" (the | means OR)

When to use Regex:

  • Matching patterns that start or end with specific text
  • Finding multiple variations (using OR: |)
  • Matching numbers or specific character types
  • Complex matching rules

💡 Tip: If you just need to match exact text, you don't need regex - use "Match Whole Word Only" instead. Regex is for complex patterns.

What is "Match Whole Word Only"?

"Whole Word Only" means the pattern must be a complete word surrounded by boundaries (spaces, punctuation, start/end of text). It prevents partial matches.

Example with Cookie Name "_abck":

Text Found Matches? Reason
_abck ✓ Yes Exact match, surrounded by boundaries
test_abck ✗ No Connected to "test", not a complete word
_abckMore ✗ No Connected to "More", not complete word

Best for matching:

  • Exact cookie names
  • Specific header names
  • Complete class or function names

💡 Tip: Without this option, searching for "_abck" might match "_abckMore", "_abckValue", and other variations.

What is "Case Sensitive"?

"Case Sensitive" means uppercase (A, B, C...) and lowercase (a, b, c...) letters are treated as different. When disabled, it ignores case differences.

Example with Cookie Name "Akamai":

Text Found With Case Sensitive Without Case Sensitive
Akamai ✓ Match ✓ Match
akamai ✗ No match ✓ Match
AKAMAI ✗ No match ✓ Match

Best for detecting:

  • Specific browser headers (often case-sensitive)
  • Cookie names with exact capitalization
  • JavaScript property names

⚠️ Important: Most browser headers and cookies ARE case-sensitive, so enable this option for accurate detection.
